# Urban-Mobility-Intelligence-Uber-Analysis
Uber-Hotspot-Detection-and-Demand-Prediction
Analyzed Uber pickup data using Python to uncover demand patterns, peak hours, and high-density locations with EDA and clustering

Objectives
- Analyze temporal patterns in Uber demand
- Identify peak hours and high-demand days
- Detect geographic hotspots using clustering
- Build a machine learning model to predict ride demand
- Generate business insights for decision-making

Day 1 Work
- Data loading and inspection
- Data cleaning
- Converted Date/Time to datetime format
- Feature engineering (hour, day, weekday)
- Basic exploratory data analysis
- Initial visualizations
- Day 2 – EDA & Time-Based Analysis

Day 2 Work
- Created time-based features: `hour`, `weekday`
- Analyzed ride demand by hour and weekday
- Built visualizations including:
  - Rides per Hour
  - Rides per Weekday
  - Hour vs Weekday Heatmap
### Key Insights
- Peak demand occurs during **4 PM – 7 PM**
- Weekdays show **strong commute-based spikes**
- Weekends have **more evenly distributed demand**
- Heatmap reveals **high activity during evening hours across all days**
